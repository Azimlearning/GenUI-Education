## 1. Overview

Codex Community Hackathon KL 2026 is a community-led AI build sprint that brings together students, founders, engineers, designers, researchers, operators, and domain experts to build practical AI products for real Malaysian problems. Unlike conventional hackathons, this is a **selected-cohort event** — up to 150 participants are chosen by the organising team based on application review, ensuring higher-quality mentoring, support, and team balance.

The event spans one week (11–18 July 2026), combining an online preparation round with an in-person build and demo day at **Sunway University, Petaling Jaya, Selangor**. The emphasis is on shipping **working demos for real users**, not polished pitch decks or generic AI showcases.

> This is a community-led event. References to Codex or AI tools do not imply third-party sponsorship, endorsement, or official status unless confirmed in writing.

## 2. Theme

### "Raising the Floor: AI for Malaysia Boleh 🇲🇾"

The hackathon is explicitly **Malaysia-first** in orientation. Teams are expected to:

- Identify **specific Malaysian users and workflows** as the problem anchor
- Build **practical AI products**, not proofs of concept or generic demos
- Demonstrate **responsible and inclusive AI use**
- Produce **working demos** that can be tested, understood, and continued after the event

The judging panel explicitly rewards clarity of local problem framing over technical novelty alone.

## 11. Project: Synapse

> Synapse is submitted to both **Codex Community Hackathon KL 2026** and **hackAstone 2026**, adapted to each competition's theme. For Codex KL, the framing emphasises the Malaysian SPM education problem and local user context. For hackAstone, the framing emphasises agentic AI pedagogy and academic innovation.

### Target User
Malaysian secondary school students (Form 4 and Form 5) in the science stream preparing for **KSSM SPM examinations** (Physics, Chemistry, Biology), and secondary school science teachers.

### The Problem

**Abstract Concept Mastery & Resource Constraints:**
The KSSM SPM science syllabus requires students to visualise abstract microscopic processes and physical experiments (e.g., osmotic pressure, chemical bonding, stoichiometry). Underfunded schools — especially in rural or B40 communities — often lack well-equipped physical labs, leaving students to memorise experiments from static textbook diagrams.

**Unaddressed Conceptual Misconceptions:**
Malaysian science students struggle with persistent, documented misconceptions (e.g., misinterpreting concentration gradients in osmosis). Generic AI chatbots and static tools do not diagnose these specific errors — they output walls of text students copy without real comprehension.

### The Solution
Synapse is an **adaptive, agentic science learning platform**. It uses a multi-agent orchestration pipeline (powered by **LangGraph**) consisting of:

| Agent | Role |
|-------|------|
| **Diagnostician** | Identifies student's specific conceptual misconception |
| **Pedagogy Strategist** | Selects the right teaching strategy (e.g., contrasting cases, predict-observe-explain) |
| **Component Composer** | Selects and configures interactive sandboxes from the pre-built component library |
| **Tutor Loop** | Maintains persistent learner memory and handles spaced repetition scheduling |

Instead of generating UI from scratch, Synapse pulls from a **pre-built, domain-correct library of parameterised components** mapped to SPM topics (e.g., diffusion, ionic vs covalent bonding, cell structure).

### Working Demo Description
The demo features a visible agent pipeline. When a student inputs a common misconception (e.g., *"osmosis is when water moves to where there is more water"*), the UI displays agents reasoning aloud. It then composes an interactive **Osmosis Simulation sandbox** where the student:
1. Makes a prediction
2. Drags a solute concentration slider
3. Runs the simulation
4. Visually observes that water moves toward higher solute concentration — contradicting and correcting their misconception

The Tutor Loop records this interaction to the learner's profile for spaced repetition. Additional demo components include an ionic/covalent bonding walkthrough, a cell structure labeler, and an equation balancer.

### Tech Stack (Planned)
- **Frontend:** Next.js / Tailwind CSS
- **Agent Orchestration:** LangGraph
- **Streaming:** Server-Sent Events (SSE) for Generative UI (typed component blocks)
- **Backend:** FastAPI (or equivalent)
- **Learner State:** Firebase (persistent learner profile)

### Post-Hackathon Intent
- Continue as Azim's FYP at Universiti Teknologi PETRONAS (UTP)
- Conduct user evaluation studies (10–15 participants)
- Potential launch as open-source or SaaS interactive learning tool for Malaysian secondary school students

## 12. Build Strategy & Key Notes

### Why Synapse Fits This Hackathon
- **Malaysian problem, Malaysian users** — directly targets Form 4/5 SPM students, one of the largest exam-prep cohorts in Malaysia
- **B40 and rural equity angle** — addresses the lab equipment gap in underfunded schools; strong "raising the floor" narrative alignment
- **Education Access track** — perfect track fit; no stretch required
- **Multilingual relevance** — future scope for Bahasa Malaysia interface strengthens the local angle

### Judging Criterion Mapping

| Criterion | Synapse Angle |
|-----------|--------------|
| Clarity of Malaysian Problem | SPM KSSM syllabus + B40 school lab gap — very specific and documented |
| Usefulness of Working Demo | Interactive misconception correction with visible agent reasoning |
| Product Design & User Workflow | Predict-observe-explain cycle; intuitive slider-based simulations |
| Responsible AI | Pedagogically grounded; no hallucinated content — components are pre-built and domain-correct |
| Technical Execution with Codex | LangGraph multi-agent pipeline + SSE Generative UI; Codex used to accelerate component scaffolding |
| Marketability & Continuation | FYP continuation plan; SaaS or open-source pathway; scalable to full SPM syllabus |
| Presentation Quality | Visible agent pipeline as a demo hook; live simulation gives strong "wow moment" |

### Build Priorities for the Sprint (11–17 July)
1. Lock the LangGraph agent pipeline (Diagnostician → Pedagogy Strategist → Component Composer → Tutor Loop)
2. SSE streaming of component blocks into the Next.js frontend
3. At minimum 3 working interactive components: Osmosis sandbox, ionic/covalent bonding walkthrough, equation balancer
4. Learner profile persistence in Firebase
5. Visible agent reasoning panel in the UI (differentiator vs generic chatbots)

### Risks & Mitigations
| Risk | Mitigation |
|------|-----------|
| Agent latency > 3s time-to-first-token | Optimise with streaming SSE; use lighter models for fast diagnostics |
| Component library too sparse for demo | Pre-build 3 high-impact components; depth over breadth |
| Cohort selection not guaranteed | Application submitted with strong, specific problem framing and existing codebase reference |
| Same project as hackAstone — perceived overlap | Both are independent submissions; Codex KL emphasises local user context, hackAstone emphasises pedagogical novelty |
